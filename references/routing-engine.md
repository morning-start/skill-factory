# Routing Engine — 路由引擎详细设计

> **所属**: Skill Factory v2.2 (Router Edition)
> **用途**: 定义智能路由表的设计原则、匹配算法和歧义处理策略
> **最后更新**: 2026-05-30

---

## 设计原则

### 1. 单一意图匹配

每个用户请求应映射到一个子技能。路由表基于关键词匹配：

| 用户输入关键词 | → 子技能 |
|--------------|---------|
| 创建/新建/从零/写一个 | Creator |
| 优化/加工/改进/精简/美化 | Processor |
| 检查/审计/合规/评分 | Processor |
| 发布/提交/版本/tag | Publisher |
| 合并/拆分/整合/重组 | Assembler |

### 2. 复合场景流水线

当用户请求包含多个意图时，按依赖顺序执行：

```
创建并发布 → creator → publisher
优化后发布 → processor → publisher
拆分后逐一发布 → assembler → creator(×N) → publisher(×N)
全量审计 → processor(逐个审计)
```

### 3. 歧义处理策略

当无法唯一确定用户意图时：

1. **列出租选项**：提供 a/b/c 菜单让用户选择
2. **追问澄清**：确认具体需求后再路由
3. **默认降级**：未匹配时导向 creator（创建是最常见需求）

## 匹配算法

### 优先级规则

```
完全匹配 (精确关键词) > 部分匹配 (同义词) > 模糊匹配 (语义近似)
```

### 关键词权重

| 权重 | 类型 | 示例 |
|------|------|------|
| 1.0 | 精确动词 | 创建/优化/发布/合并 |
| 0.8 | 同义词 | 新建/改进/提交/整合 |
| 0.5 | 名词短语 | 新技能/质量检查/版本管理 |

## 验证方法

路由正确性的验证通过 `tests/scenarios/skill-factory-root/scenarios.yaml` 中的 20 个压力测试场景进行，覆盖：
- 单一意图路由（10 场景）
- 复合场景流水线（5 场景）
- 歧义处理（3 场景）
- 边界条件（2 场景）
