## 📋 核心任务

### 任务 1: 技能依赖图可视化工具 (12d)

#### Step 1: Dependency Parser (4d)
- 解析 metadata.json 的 `dependency` 字段
- 构建内存中的依赖关系图（有向无环图检测）
- 支持递归解析子技能的依赖

```python
class DependencyGraph:
    def parse(self, metadata_path: str) -> DiGraph:
        """从 metadata.json 构建依赖图"""
        
    def detect_cycles(self) -> List[Cycle]:
        """检测循环依赖"""
        
    def get_layer(self, skill_name: str) -> int:
        """计算层级深度"""
```

#### Step 2: Mermaid Generator (5d)
- 将 DiGraph 转换为 Mermaid graph TD 语法
- 支持多种布局选项（TB/LR/RL）
- 自动着色（按阶段区分颜色）
- 嵌入到 SKILL.md 或独立输出

```bash
$ skill-factory dependency-graph --format mermaid --layout LR

# 输出
graph LR
    SF[skill-factory] --> R[researcher]
    R --> A[analyzer]
    ...
```

#### Step 3: Interactive Viewer (3d)
- Web 界面展示依赖图（可选）
- 支持点击节点查看详情
- 导出 PNG/SVG 功能

### 任务 2: Diff 预览功能 (8d)

#### Step 1: Diff Engine (4d)
- 实现 Markdown 文本 diff 算法
- 忽略空白字符差异
- 生成结构化变更报告

```yaml
diff_report:
  file: skills/my-skill/SKILL.md
  operation: enricher.add_examples
  
statistics:
  lines_added: 45
  lines_removed: 12
  lines_modified: 8
  net_change: +33 (+12%)
  
changes:
  - type: addition
    location: "L150-L180"
    content: |
      + ## 使用示例
      +
      + ### 示例 3: 处理大型仓库
      + ```bash
      + $ skill-factory generate --project ./monorepo --type 4
      + ```
      
  - type: modification
    location: "L45"
    old: "description: '生成技能文件'"
    new: "description: '生成符合规范的技能包'"
```

#### Step 2: 集成到加工器 (2d)
- Enricher 输出时自动附带 diff
- Simplifier 输出时自动附带 diff
- 用户确认后才实际应用修改

#### Step 2: CLI 集成 (2d)
```bash
$ skill-factory enrich --dry-run --diff my-skill/

# 显示预览，不实际修改
# 用户确认后执行:
$ skill-factory enrich --apply my-skill/
```

### 任务 4: WORKFLOW 支持完善 (5d)

#### Step 1: 定义 WORKFLOW 触发条件 (2d)

```yaml
workflow_generation_rules:
  #何时应该生成 WORKFLOW.md？
  triggers:
    - skill_type: [3, 4]  # 轻+厚 或 重+厚
    - has_multiple_steps: true  # 包含多个顺序步骤
    - user_request: explicit     # 用户明确要求
    
  #何时不应该生成？
  skip_when:
    - skill_type: 1  # 简单技能不需要
    - single_action: true  # 单一操作
```

#### Step 2: WORKFLOW 模板标准化 (2d)

创建标准模板：

```markdown
# {skill-name} Workflow

## Quick Start (5min)
最快的上手方式...

## Standard Flow (30min)
推荐的标准使用流程...

## Advanced Usage (2h)
高级功能和配置...

## Troubleshooting
常见问题和解决方案...
```

#### Step 3: Generator 集成 (1d)

Generator 在生成 Type 3/4 时自动创建 WORKFLOW.md 骨架。
